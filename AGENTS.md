# Repository Guidelines

## 项目概览

面向 Java 开发者的 LangChain v1 学习仓库。代码按阶段递进组织，每个阶段沉淀笔记、手写骨架和练习代码。

- **LangChain**: `1.3.9`、**LangGraph**: `1.2.5`、**Python**: `>=3.11`

## 协作约束

- **代码注释与 Git 提交信息**：尽量使用中文（技术术语用英文）。建议提交信息使用“标题 + 正文要点”格式：标题形如 `LC-xx：中文说明`；正文用 `-` 列出关键改动，避免只给一句过短标题。
- **不执行 `git commit`**：只给出建议提交信息，由用户自行审核并手动提交。普通代码检查、只读诊断和必要验证命令可按任务需要执行。
- **阶段启动少跑命令**：开始新学习阶段时，默认只做必要的文档、骨架和计划更新；不主动执行 `uv` 验证或 `git status` 等额外诊断，除非用户明确要求或任务确有必要。
- **引导而非代劳**：遇到环境、依赖、长期配置等需要用户掌握或确认的问题时，优先引导用户手动处理，不主动代执行。
- **最小改动原则**：不随意重写用户已有代码。确需重构时先说明原因并征询意见。必须修改原代码时，优先注释保留旧写法而不是直接删除。
- **PowerShell 中文编码**：在 PowerShell 中读取中文文档时，必须显式使用 UTF-8；优先用 `[Text.Encoding]::UTF8.GetString([IO.File]::ReadAllBytes('文件路径'))` 定向读取，避免先用默认编码读出乱码并污染上下文。
- **规则沉淀**：记录用户新增约束时，尽量将用户原话提炼为更简洁、明确的规则。

## 上下文管理

- 进入新阶段时，优先定向读取 `.agents/LANGCHAIN_PLAN.md` 中的“Codex Agent 执行说明”和当前阶段行，确认执行规则、当前任务和产出要求。
- 新阶段启动时应窄读上下文，再按需定向读取 `.agents/STAGE_SUMMARIES.md` 中的上一阶段摘要。
- 回顾历史阶段时，只有当前任务明确依赖历史细节才补读：优先读取 `.agents/STAGE_SUMMARIES.md` 中的对应阶段摘要；摘要不足时先看对应阶段目录结构或骨架文件开头/TODO 小段；仍不足再打开阶段文档的相关小节，避免直接读取完整阶段文档。
- 官方文档核对只提取当前知识点必要的 API、导入路径和核心概念；避免一次性打开多页长文档或无关页面。若必须扩大读取范围，先说明原因。

## 文档职责边界

- `AGENTS.md`：只记录长期稳定的协作规则、流程约束、上下文管理策略。
- `.agents/LANGCHAIN_PLAN.md`：记录学习路线、阶段状态、推进备注和 Codex Agent 执行说明；新阶段启动时需定向读取执行说明。
- `.agents/STAGE_SUMMARIES.md`：记录各阶段轻量摘要；上下文恢复时按阶段定向读取，不默认全文读取。
- `.agents/CHANGE_LOG.md`：记录计划变更历史，仅用于审计计划演进；新阶段启动和普通上下文恢复时不读取。
- `learning/*/*.md`：记录具体阶段的完整学习笔记、API 细节和实践复盘。
- `*_skeleton.py`：保留学习者手写练习代码。
- `*_skeleton.origin.py`：保留初始骨架副本，作为对照。
