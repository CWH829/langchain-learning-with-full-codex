# Repository Guidelines

## 项目概览

面向 Java 开发者的 LangChain v1 学习仓库。代码按阶段递进组织，每个阶段沉淀笔记、手写骨架和练习代码。

- **LangChain**: `1.3.9`、**LangGraph**: `1.2.5`、**Python**: `>=3.11`

## 协作约束

- **代码注释与 Git 提交信息**：尽量使用中文（技术术语用英文）。建议格式：`LC-xx：中文说明`。
- **不执行 `git commit`**：只给出建议提交信息，由用户自行审核并手动提交。普通代码检查、只读诊断和必要验证命令可按任务需要执行。
- **引导而非代劳**：遇到环境、依赖、长期配置等需要用户掌握或确认的问题时，优先引导用户手动处理，不主动代执行。
- **最小改动原则**：不随意重写用户已有代码。确需重构时先说明原因并征询意见。必须修改原代码时，优先注释保留旧写法而不是直接删除。
- **PowerShell 中文编码**：在 PowerShell 中读取中文文档时，显式设置 UTF-8 输出，避免乱码。
- **规则沉淀**：记录用户新增约束时，尽量将用户原话提炼为更简洁、明确的规则。

## 上下文管理

- 进入新阶段时，优先读取 `.agents/LANGCHAIN_PLAN.md`，确认当前任务、产出要求和必要前置摘要。
- 除非当前任务明确依赖历史细节，不默认回读所有已完成阶段全文。
- 需要回顾历史阶段时，优先读取 `.agents/LANGCHAIN_PLAN.md` 中的阶段摘要；只有摘要不足时，再打开对应阶段完整文档。

## 文档职责边界

- `AGENTS.md`：只记录长期稳定的协作规则、流程约束、上下文管理策略。
- `.agents/LANGCHAIN_PLAN.md`：记录学习路线、阶段状态、阶段摘要和推进备注。
- `learning/*/*.md`：记录具体阶段的完整学习笔记、API 细节和实践复盘。
- `*_skeleton.py`：保留学习者手写练习代码。
- `*_skeleton.origin.py`：保留初始骨架副本，作为对照。
